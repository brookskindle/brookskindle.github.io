Favicons

  • Tiger Oakes
  • IE5 introduced the favicon (portmanteau of favorites and icon)
  • can include favicons with a <link rel="icon" ... /> tag
  • Progressive Web Apps (PWAs) use a different syntax for specifying favicons, since the favicon can be used for many more scenarios than just desktop/web.
  • favicons no longer have to be in .ico format, they can be in .png
  • there are tons of "normal" favicon sizes: 16x16, 32x32, 45x45, 57x57, 72x72, 114x114, 128x128, and even 270x270 and 558x558. Those are huge!
    • https://github.com/audreyfeldroy/favicon-cheat-sheet
  • favicons can also use vector graphics (.svg) which can allow image scaling without having to hand-create multiple favicon files
  • @media query selector allows for light/dark mode favicon swapping, as well as high/low contrast
  • what are maskable icons? Android apparently used to allow non-square/circle icon shapes
  • what are monochrome icons for PWAs?
  • https://maskable.app
  • https://monochrome.fyi
  • recommends at least a 48x48 icon for your favicon, if you only go with one

i18n/i10n -- Internationalization

  • Romulo Cintra
  • goal: make the web more familiar to everyone
  • screen readers get fucked up trying to translate things if i18n isn't followed
  • can localize fonts and quotation marks using pure CSS (i18n support)
  • Intl is a global javascript object available in the browser, to help with so so so many common internationalization issues

Accessibility as a Concept // Jemima Abu

  • Not everyone uses things as you do. Not everyone can.
  • https://increment.com, blog on internationalization
  • lighthouse is a chrome developer tool that helps test accessibility
  • eslint-plugin-jsx-a11y is an accessibility/ally lint checker
  • chrome vox is a chrome extension that lets you experience sites as a screen reader

Memory leaks in JS // Will Klein

  • js has a stack & heap, just like traditional programming languages
  • primitives go on stack (size can be pre-determined), objects go on heap (dynamic in size)
  • chrome dev tools --> memory --> heap snapshot is your friend
  • firefox also has a great heap visualization tool
  • lru-cache, and performance.measureMemory are useful tools

Saving data to the decentralized web // Kelsey Breseman

  • https://envirodatagov.org/publications
  • Problem: people assume the internet is forever, but that is not the case
    • things get taken down, often important things we should have access to
    • conflicts of interest / political agendas are also at play here
  • Solution? decentralizing the web
  • IPFS
  • uses file chunks + metadata to calculate hash and ensure correctness
  • "Data is considered to be at risk unless [data owners] have a dedicated plan to not be at risk"
  • published paper: Risk Assessment for Scientific Data
  • For the data/privacy conscious, check out: brave, ipfs, mastadon, beaker, and scuttlebutt.nz
